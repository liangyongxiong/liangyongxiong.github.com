使用 Pelican + Github 搭建静态博客

技术 Python / Pelican / Github 评论
Michael 2013-11-06
Pelican 是一套开源的使用 Python 编写的静态博客生成框架，可以添加文章和和创建页面，使用 MarkDown 格式撰写文章内容，同时支持 Disqus 评论系统、RSS 和 Atom 输出、插件、主题，代码高亮等功能，采用 Jajin2 模板引擎，可以很容易的更改模板。

介绍 Pelican

Pelican[ˈpɛlɪkən] 是法语 Calepin[kalp ɛ̃]变形词（anagram），Calepin 是笔记本、备忘录的意思。

框架选型上，我有以下考虑，经历各种 Google + Baidu 后，最终确定了 Pelican + Github 的方案。

  • WordPress或Discuz!都基于 PHP 语言开发，Pythoner 略微抵触
  • Django框架虽然功能完善，但需要配置和管理的相应的数据库等组件，过于重量级
  • 技术宅都喜欢免费的午餐，不希望支付VPS服务器的月租费用


初始化 Pelican 项目

在 Python + Pip + Ubuntu 环境下，直接通过pip install命令，即可安装 Pelican

sudo pip install pelican    # 安装 Pelican 框架
sudo pip install markdown   # 安装 MarkDown 组件

创建博客文件夹，并执行初始化命令

mkdir /path/to/your/blog
cd /path/to/your/blog
pelican-quickstart          # 生成 Pelican 项目文件框架

回答一系列问题过后，就能生成以下博客项目文件

/path/to/your/blog
  |-- content               # 全部文章放于此目录
  |-- output                # 静态文件放于此目录
  |-- pelicanconf.py        # 博客配置文件
  |-- publishconf.py        # 发布配置文件
  |-- Makefile              # 方便管理博客的 Makefile
  |-- develop_server.sh     # 用于开启测试服务器


编辑博客文章

在content目录中新建一个test.md文件，输入以下内容

Title: Hello World
Date: 2015-01-11 10:30
Category: 分类1
Tags: 标签1,标签2

Hello World! 
这是我的第一篇文章
欢迎光临

在博客项目文件夹中执行以下指令，生成全部静态页面文件

cd /path/to/your/blog
make html

本地开启一个测试服务器, 这会在本地8000端口建立一个测试web服务器, 可以使用浏览器打开http://localhost:8000/来访问这个测试服务器, 然后就可以欣赏到你的博客了

cd /path/to/your/blog
./develop_server.sh start


提交到 Github

创建 Github 工程，工程名称必须为username.github.com，其中username是你在 Github 上注册的用户名，并将 Github 工程clone到 Pelican 项目的output文件夹中。

cd /path/to/your/blog
git clone https://github.com/username/username.github.com.git ./output

在output目录中commit全部文件到 Github

cd outout
git add --all
git commit -m 'first commit'
git push


配置 Pelican 项目

Pelican 提供了一些主题和插件，用户可从 Github 克隆下来，然后根据需要自行选择

cd /path/to/your/blog
git clone git://github.com/getpelican/pelican-themes.git    # 主题
git clone git://github.com/getpelican/pelican-plugins.git   # 插件
